## Опис
У цьому проєкті реалізовано бібліотеку функцій вищого порядку для трансформації даних.  
Усі основні функції написані вручну без використання готових аналогів для основної логіки.

## Реалізовані функції

### 1. `myMap(fn, array)`
Власна реалізація методу `map`.  
Повертає новий масив, у якому кожен елемент є результатом виклику функції `fn`.

### 2. `myFilter(fn, array)`
Власна реалізація методу `filter`.  
Повертає новий масив лише з тих елементів, які задовольняють умову.

### 3. `myReduce(fn, initialValue, array)`
Власна реалізація методу `reduce`.  
Послідовно об’єднує елементи масиву в одне значення.

### 4. `compose(...fns)`
Композиція функцій справа наліво.  
`compose(f, g, h)(x)` → `f(g(h(x)))`

### 5. `pipe(...fns)`
Композиція функцій зліва направо.  
`pipe(f, g, h)(x)` → `h(g(f(x)))`

### 6. `curry(fn)`
Реалізація каррінгу довільної функції.  
Дозволяє викликати функцію частинами.

### 7. `partial(fn, ...fixedArgs)`
Часткове застосування функції.  
Дає можливість зафіксувати частину аргументів наперед.

### 8. `memoize(fn)`
Мемоізація результатів функції.  
Кешує результат, щоб не виконувати обчислення повторно.

### 9. `chain(data)`
Ланцюжковий API для послідовної обробки даних.
